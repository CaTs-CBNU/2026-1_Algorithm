# 하노이의 탑 풀이 과정
## 1. Case
### 기본적으로 하노이의 탑은 다음과 같이 이동한다.
1. 1 -> 2 (보조탑으로의 가장 작은 원판의 이동)
2. 1 -> 3 (목적지 탑으로의 그 다음 원판의 이동)
3. 2 -> 3 (목적지 탑으로의 가장 작은 원판의 이동)
4. 1 -> 2(보조 탑으로의 그 다음 원판의 이동)
5. 3 -> 2(보조 탑으로의 가장 작은 원판의 이동)
6. 3 -> 1(시작 탑으로의 그 다음 원판의 이동)
7. 2 -> 1(시작 탑으로의 가장 작은 원판의 이동)
8. ...
### 문제의 요약
1. 어느 탑으로 갈지 정해야 한다.
- 시작 : start
- 보조 : via
- 목적지 : end
2. 원판의 개수는 중요하지 않다.
- 원판의 개수 : n
3. 경향을 파악해야 한다.
- n이 움직일 때마다 1씩 감소
- 움직임을 start -> end 라고 정의
- 배열에 넣는 동작을 움직임이라고 정의
### 구조에 따른 구현
1. 하노이의 탑의 문제의 구조
    1. 위에 있는 n-1개를 start에서 via로 이동
    2. 가장 큰 원판 1개를 start에서 end로 이동
    3. via에 있던 n-1개를 다시 end로 이동
    ```
    hanoi(n-1, start, via, end)
    answer.append([start, end])
    hanoi(n-1, via, end, start)
    ```
2. 재귀를 멈추는 조건 정의
    n==1이면 더 이상 이동할 필요가 없음 ->
    ```
    if n == 1: 
        answer.append([start, end])
        return
    ```
3. 시작 : hanoi(n, start, end, via)
